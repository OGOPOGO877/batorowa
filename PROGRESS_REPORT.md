# batorowa-project 現在の進捗状況レポート
(他のAIへの共有用)

## プロジェクト概要
- Phaser 3 を使ったトップダウン視点のバトロワ風サバイバルゲーム
- モジュラー構成:
  - index.html + js/main.js (本番メインループ、移植先)
  - js/player.js (視界、装備効果、NPC視認)
  - js/npc.js, js/audio.js, js/combat.js, js/ui.js, js/world.js, js/config.js
  - test.html (練習モード。変更のプロトタイプ場として使用 → 本番に移植)
- アセットパスは相対 (assets/sounds/..., assets/images/armor/goggles.png など)
- ローカルサーバー必須 (python -m http.server)。file://だと音/画像が読めない

## 現在の主な実装済み機能 (本番側に最新状態を移植済み)
### 操作
- WASD: 移動
- マウス: 視界方向 (扇形コーン + LOS + 暗闇オーバーレイ)
- **左クリック**: 射撃のみ (スペースは拾い専用。同時押し問題を解消)
- **スペース**: 範囲内アイテムを拾う (自動拾い完全廃止)
- F8: デバッグビジュアルトグル
- F9: リスタート (デバッグ時のみ)

### 視界・検知システム
- プレイヤー視界: マウス指向のコーン。壁でLOSチェックあり。ヒステリシスでチラつき軽減。
- **ゴーグル装備時 (visionMultiplier: 1.4)**:
  - コーンの描画サイズが大きくなる
  - 実際の視認範囲 (NPCが見える距離) もそれに合わせて拡大 (player.js の updateNPCVisibility で getEffectiveVisionRadius を使用)
  - ゴーグル未装備時は従来の固定 VISION_RADIUS (520) のまま
- 自キャラ: 常に明るく表示 (setTint(0xffffff) + alpha=1.0 + 深度高め)。視覚範囲内でも埋もれない
- NPC: 各自の視界 + 音検知 (足音/銃声で investigating など)
- 音は視界外でも聞こえる (減衰 + 方向インジケータ)

### 足音SE (walk.wav)
- 移動中 (pSpeed > 10) のみ
- 0.35秒 (350ms) 間隔で1回試行
- 既に再生中なら重複再生しない (isPlaying チェック)
- 停止したら即停止
- デバッグ用に仮想イベント (黄色円) は残す (AIは「一旦足音拾わない」設定)
- 目的: リアルなループではなく、ゲーム的な「ザッ (0.35s) ザッ ...」表現

### 拾い・装備システム (最新仕様)
- 自動拾い完全廃止
- 取得範囲: 60px
- Space押下時のみ、範囲内の最寄りアイテムを拾う
- 武器を拾うとき、現在持ってる武器と**違う種類**なら、その場にドロップ (新しいlootアイテム生成)
  - ドロップ時はキー マッピング (pistol → hg など) でビジュアル生成
- 装備効果: ゴーグル (視界拡大)、アーマー (ダメージ軽減)、ヘッドセット (聴覚) など

### 射撃・武器
- データ駆動 (WEAPONS_DATA でリロード時間、マガジン、拡散など)
- 左クリック射撃 + リロード管理
- 異なる武器拾い時のドロップ対応

### UI
- 左下 (HP / WEAPON / AMMO / GEAR):
  - フォント拡大 (HP 22px、WEAPON/AMMO 18px、GEAR 16px)
  - 半透明パネル + 細い青枠で「かっこよく」
  - 全体を約50px上方にシフト
- 右上: ALIVE 表示
- ゴーグル装備時は GEAR に "GOGGLES" とテキスト表示

### その他
- **カスタムレティクル**: システムカーソル完全非表示 + 自前で黄色い円＋十字をマウス位置に描画 (ワールド座標追従)
- デバッグ (F8): NPCの青視界扇、シアン聴覚円、黄色足音円、赤銃声円、状態テキスト
- ゴーグル.png: 正しいパス (assets/images/armor/goggles.png) に修正済み。lootアイテムが画像で表示される
- ゾーン収縮、勝敗 (最後の生存者) などの基本システム
